> 전국 주소를 표준 형식으로 변환하는 도구 (웹 + Chrome 익스텐션)

**[웹에서 사용하기 (gjdong.vercel.app)](https://gjdong.vercel.app)**

## 이런 업무에 활용하세요

### 복지/주민지원 업무
> "취약계층 100가구 현장방문 해야 하는데, 주소가 뒤죽박죽이에요"

민원인이 작성한 주소 → **도로명 + 지번 + 행정동** 한번에 정리, 지도에서 동선 확인

### 시설관리 업무
> "관내 경로당 200개소 주소를 행정동별로 분류해야 해요"

시설 목록 엑셀 붙여넣기 → **행정동 자동 추출** → 엑셀 다운로드

### 현장점검/단속 업무
> "불법광고물 신고 30건 위치를 한눈에 보고 싶어요"

신고 주소 일괄 입력 → **지도에 번호 마커로 표시** → 효율적 동선 계획

### 통계/보고 업무
> "사업 대상지 주소를 보고서 양식에 맞게 통일해야 해요"

다양한 형식의 주소 → **표준형식으로 일괄 변환** → 복사해서 보고서에 붙여넣기

## 주요 기능

### 1. 주소 변환
다양한 형태로 입력해도 표준 형식으로 변환됩니다.

| 입력 | 출력 (표준형식) |
|------|----------------|
| 광진구 아차산로 400 | 서울특별시 광진구 아차산로400(자양동 870, 자양2동) |
| 자양동 870 | 서울특별시 광진구 아차산로400(자양동 870, 자양2동) |
| 경기도 성남시 분당구 판교역로 235 | 경기도 성남시 분당구 판교역로235(삼평동 681, 삼평동) |

### 2. 일괄 변환
여러 주소를 한번에 변환할 수 있습니다.
- **줄바꿈**으로 구분하여 입력
- **엑셀에서 복사/붙여넣기** 지원 (시설명 + 주소 2열 형식)
- 결과를 **엑셀 파일로 다운로드**

### 3. 지도 표시
변환된 주소의 위치를 지도에서 확인할 수 있습니다.
- 단일 주소: 해당 위치에 마커 표시
- 일괄 주소: 모든 위치에 번호 마커 표시

### 4. Tableau Geocoder
CSV/엑셀 파일을 업로드하면 위도/경도 컬럼을 추가해줍니다.
- `/tableau-geocoder` 페이지에서 사용
- Tableau, Power BI 등 BI 도구에서 지도 시각화용

### 5. Chrome 익스텐션
브라우저에서 바로 주소를 변환할 수 있는 확장 프로그램입니다.
- **팝업 변환**: 익스텐션 아이콘 클릭 → 단일/일괄 주소 변환
- **우클릭 변환**: 웹페이지에서 주소 드래그 → 우클릭 → "표준주소 변환"
- **단축키 변환**: `Ctrl+Shift+C` (Mac: `Cmd+Shift+C`) → 클립보드 주소 즉시 변환
- **자동 감지**: 웹페이지에서 주소 복사 시 자동으로 변환 (설정에서 활성화)
- **7가지 출력 포맷**: 표준형식, 도로명, 지번, 행정동, 우편번호, 세부주소
- **즐겨찾기/히스토리**: 최근 변환 기록 저장 및 즐겨찾기

## 사용 방법

### 단일 주소 변환
1. 주소 입력 (예: `광진구 아차산로 400`)
2. **주소 검색** 버튼 클릭
3. 변환된 결과 클릭하여 복사

### 일괄 주소 변환
1. 여러 주소를 줄바꿈으로 구분하여 입력
   ```
   광진구 아차산로 400
   강남구 테헤란로 152
   부산시 해운대구 우동 1523
   ```
2. 결과 표시 방법 선택 (한번에 출력 / 개별 출력)
3. **주소 검색** 버튼 클릭
4. 결과 복사 또는 **엑셀 다운로드**

### 엑셀 데이터 붙여넣기
엑셀에서 2열 (시설명 | 주소) 형태로 복사하면 시설명도 함께 처리됩니다.

## 개발자용 정보

### 설치 및 실행

**웹 앱 (Next.js)**
```bash
npm install --legacy-peer-deps
npm run dev      # 개발 서버 (http://localhost:3000)
npm run build    # 프로덕션 빌드
```

**Chrome 익스텐션 (Plasmo)**
```bash
cd extension
npm install
npm run dev      # 개발 모드 (핫 리로드)
npm run build    # 프로덕션 빌드 (build/ 폴더)
npm run package  # .zip 패키징 (Web Store 배포용)
```

빌드 후 `chrome://extensions` → 개발자 모드 → "압축해제된 확장 프로그램을 로드합니다" → `extension/build/chrome-mv3-prod` 폴더 선택

### 환경 변수

`.env.local` 파일 생성:
```
KAKAO_REST_API_KEY=your_kakao_rest_api_key
```

**Kakao API 키 발급:**
1. [Kakao Developers](https://developers.kakao.com/) 접속
2. 애플리케이션 생성
3. 앱 키 > REST API 키 복사
4. 플랫폼 > Web 추가 > 도메인 등록

### 기술 스택

**웹 앱**
- **Framework**: Next.js 15 (App Router), React 19
- **UI**: Tailwind CSS 4, shadcn/ui
- **지도**: Leaflet.js (OpenStreetMap)
- **API**: Kakao Local API
- **엑셀**: xlsx 라이브러리

**Chrome 익스텐션**
- **Framework**: Plasmo, React 19, TypeScript
- **UI**: Tailwind CSS 3
- **저장소**: @plasmohq/storage (chrome.storage 래핑)
- **Manifest**: V3 (service worker, content script)

### Vercel 배포

1. GitHub에 푸시
2. [Vercel](https://vercel.com)에서 import
3. Environment Variables에 `KAKAO_REST_API_KEY` 추가
4. Deploy
